# Idea Capture & Refinement

아이디어를 빠르게 캡처하고, 이후 에이전트가 정리·숙성시키는 워크플로우를 담당한다.

## 캡처 (Capture)

사용자가 아이디어를 던지면 즉시 `ideas/` 폴더에 노트를 생성한다.

### 파일 생성 규칙

Obsidian CLI를 사용하여 노트를 생성한다:

```bash
obsidian create name="아이디어-제목" content="frontmatter + 본문" path="ideas/아이디어-제목.md" silent
```

파일명은 아이디어 핵심을 담은 짧은 한글 또는 영문 slug로 짓는다. 날짜 접두사는 붙이지 않는다 (frontmatter의 created 필드가 대신한다).

### Frontmatter 구조

모든 아이디어 노트는 다음 frontmatter를 포함한다:

```yaml
---
status: seed
tags: []
created: 2026-03-27
---
```

- **status**: 아이디어의 성숙도. 아래 라이프사이클 참조.
- **tags**: 관련 키워드. 캡처 시점에 에이전트가 내용을 보고 자동으로 태깅한다.
- **created**: 생성일.

태그는 아이디어 내용에서 자연스럽게 추론한다. 예시:
- 경쟁사 언급 → `경쟁사분석`
- 사용자 경험 관련 → `UX`
- 수익/비용 관련 → `비즈니스모델`
- 기술 스택 관련 → `기술`
- 시장/트렌드 관련 → `시장분석`

### 캡처 후 응답

캡처가 끝나면 간결하게 확인만 한다:

> 💡 **아이디어-제목** 저장했습니다. (status: seed, tags: #태그1 #태그2)

길게 설명하거나 되물을 필요 없다. 빠르게 저장하는 것이 핵심이다.

## 라이프사이클 (Lifecycle)

아이디어는 4단계 상태를 거친다:

| status | 의미 | 누가 전환하는가 |
|---|---|---|
| `seed` | 사용자가 던져넣은 원본. 에이전트가 내용을 건드리지 않음 | 캡처 시 자동 |
| `growing` | 에이전트가 관련 정보를 붙여 살을 붙인 상태 | 사용자 요청 시 에이전트가 전환 |
| `mature` | 충분히 정리되어 프로젝트화 검토 가능 | 에이전트가 판단하여 제안 |
| `promoted` | `projects/`로 졸업 완료 | 사용자 승인 후 에이전트가 전환 |

## 정리 (Refine)

사용자가 "아이디어 정리해줘", "이거 좀 발전시켜봐" 등을 요청하면 다음 작업을 수행한다:

1. `obsidian search query="status: seed"` 또는 특정 태그로 대상 노트를 찾는다
2. 노트 내용을 읽고 다음을 추가한다:
   - **경쟁사 분석**: 유사 서비스/제품이 있는지 조사
   - **시장 분석**: 관련 시장 규모, 트렌드
   - **위험요소**: 실행 시 예상되는 리스크
   - **실행 방안**: 구체적인 다음 단계 제안
3. status를 `growing`으로 변경한다:
   ```bash
   obsidian property:set name="status" value="growing" file="아이디어-제목"
   ```

## 승격 (Promote)

아이디어가 충분히 성숙하면 사용자에게 프로젝트화를 제안한다. 승인되면:

1. `projects/{프로젝트명}/` 구조를 rules.md의 프로젝트 구조 규칙에 따라 생성한다
2. 아이디어 노트의 내용을 프로젝트 agents.md의 기반으로 활용한다
3. 원본 아이디어 노트의 status를 `promoted`로 변경하고, 프로젝트로의 위키링크를 추가한다

## 조회 (Review)

사용자가 "아이디어 목록", "지금 아이디어 뭐있어" 등을 요청하면:

```bash
obsidian search query="status: seed OR status: growing"
obsidian tags sort=count counts
```

상태별로 그룹핑하여 간결하게 보여준다.
